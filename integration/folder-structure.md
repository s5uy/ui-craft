## Placement Rules

| Component type | Location |
|---|---|
| shadcn primitive (generated by CLI) | `components/ui/` |
| Custom extension of shadcn primitive | `components/ui/` (same file, extend CVA) |
| Feature-specific component | `components/[feature]/` |
| Layout components (navbar, sidebar, footer) | `components/layout/` |
| Used in 2+ features | `components/` (top-level) |
| Used in 1 feature only | `components/[feature]/` |
| Page component | `app/[route]/page.tsx` |
| Route layout | `app/[route]/layout.tsx` |
| Utility function | `lib/[utility].ts` |
| Custom hook | `hooks/use-[name].ts` |

---

## Route Group Conventions

Route groups (`(groupName)`) allow layouts without URL segments:

- `(auth)` вЂ” no header/sidebar, centered form layout
- `(app)` вЂ” persistent sidebar + navbar
- `(marketing)` вЂ” marketing navbar + footer, no app chrome

```tsx
// app/(app)/layout.tsx вЂ” shared layout for authenticated app routes
export default function AppLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="flex h-screen overflow-hidden">
      <Sidebar />
      <main className="flex-1 overflow-y-auto">{children}</main>
    </div>
  )
}

// app/(auth)/layout.tsx вЂ” no chrome for auth pages
export default function AuthLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="flex min-h-screen items-center justify-center bg-muted/20">
      {children}
    </div>
  )
}
```

## `lib/utils.ts`

The `cn()` utility is required:

```ts
import { type ClassValue, clsx } from "clsx"
import { twMerge } from "tailwind-merge"

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

Install if not present: `npm install clsx tailwind-merge`
