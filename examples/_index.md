# Examples — Index

Complete, working implementations that demonstrate the ui-craft skill in action. Each example shows a specific blueprint + palette combination applied to a realistic product context.

---

## Philosophy

Examples serve three purposes:
1. **Calibration** — what the output quality bar looks like in code
2. **Inspiration** — concrete, copy-able patterns for common product types
3. **Validation** — evidence that the philosophy produces good results

Few and excellent, not many and mediocre. Every example must represent the target quality level — not an average output, but a best-case output.

## How to Use Examples

1. Read the `README.md` in each example directory — it documents the design decisions
2. The design decisions (what was chosen and why it was avoided) are as valuable as the code
3. Reference the pattern for similar product contexts — don't copy verbatim
